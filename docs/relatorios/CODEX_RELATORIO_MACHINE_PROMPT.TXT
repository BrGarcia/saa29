SAA29 review context

Scope
- Reviewed against docs\requirements\00_SRS.md and docs\requirements\01_SPECS.md.
- Focus: correctness, security, architecture.

Primary findings
1. HIGH: unauthenticated user creation.
- File: app\auth\router.py
- POST /auth/usuarios has no auth/role check.
- Risk: arbitrary account creation and access bypass.
- Fix: require authenticated privileged user.

2. HIGH: public attachment exposure.
- File: app\main.py
- /uploads served via StaticFiles without authorization.
- Risk: unauthorized access to pane evidence/files.
- Fix: remove public static serving; use authenticated download endpoint.

3. HIGH: weak upload validation.
- File: app\panes\service.py
- Validation uses extension only; ignores MIME/content signature; accepts .pdf although SPECS says jpg/png.
- Risk: malicious disguised files + spec drift.
- Fix: validate MIME + magic bytes; align allowed types with spec or formally update spec.

4. MEDIUM: pane resolution can miss resolver identity.
- File: app\panes\service.py
- PUT /panes/{id} can set RESOLVIDA with data_conclusao but without concluido_por_id.
- Risk: broken audit trail/rastreability.
- Fix: force dedicated conclude flow or set concluido_por_id in all resolution paths.

5. MEDIUM: date filters not exposed in API.
- Files: app\panes\service.py, app\panes\router.py
- Service supports data_inicio/data_fim; router does not.
- Risk: RF-06 only partially implemented.
- Fix: add date query params to GET /panes/.

6. MEDIUM: upload route behavior depends on startup ordering.
- File: app\main.py
- upload dir is created in startup, but static mount happens before startup.
- Risk: clean deploy may not mount /uploads as expected.
- Fix: create directory before mount or remove static serving approach.

7. MEDIUM: inconsistent HTTP error semantics.
- Files: routers in aeronaves/equipamentos/panes.
- 404/409 mapping mixes not-found and business-conflict cases.
- Risk: unreliable client behavior and harder integrations.
- Fix: standardize 404 not found, 409 conflict, 422 invalid input.

8. LOW/MEDIUM: implicit transaction boundary.
- File: app\dependencies.py
- get_db auto-commits at request end.
- Risk: hidden transaction behavior and harder maintenance.
- Fix: document clearly or move transaction control to explicit service/application layer.

9. LOW/MEDIUM: code/spec mismatch.
- Examples: PDF allowed in upload though SPECS says jpg/png; observacao_conclusao accepted but not persisted.
- Risk: maintenance confusion and misleading contracts.
- Fix: align docs, API contracts, and persistence behavior.

10. LOW: test gaps on critical risks.
- Missing tests for protected user creation, unauthorized attachment access, concluido_por_id consistency, date filters, and real file validation.

Recommended review priority
1. Auth on user creation
2. Attachment exposure
3. Upload validation
4. Pane conclusion audit consistency
5. Date filter API gap
6. HTTP error normalization
7. Test hardening

Verification note
- Findings are from code review. Test execution was attempted but timed out in current environment.
