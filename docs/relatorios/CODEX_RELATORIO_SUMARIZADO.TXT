CODEX RELATORIO SUMARIZADO - SAA29
Data: 2026-03-28

Escopo
- Revisao baseada em docs\requirements\00_SRS.md, docs\requirements\01_SPECS.md e codigo dos modulos auth, panes, aeronaves e equipamentos.
- Foco: corretude, seguranca, arquitetura.

Resumo
- Problemas principais em controle de acesso, uploads, rastreabilidade de conclusao de panes, aderencia parcial aos requisitos e semantica de erros HTTP.

Achados principais

1. Alta
Criacao de usuarios sem autenticacao
- Arquivo: app\auth\router.py
- POST /auth/usuarios nao exige CurrentUser.
- Risco: qualquer cliente pode criar conta valida.
- Fix: exigir autenticacao e autorizacao por perfil.

2. Alta
Uploads expostos publicamente
- Arquivo: app\main.py
- /uploads e servido via StaticFiles sem auth.
- Risco: vazamento de anexos e evidencias.
- Fix: servir anexos via endpoint autenticado e autorizado.

3. Alta
Validacao de upload fraca
- Arquivo: app\panes\service.py
- Validacao so por extensao; tipo_mime ignorado; aceita .pdf fora da SPECS.
- Risco: arquivo malicioso disfarçado e desalinhamento com requisito.
- Fix: validar MIME + assinatura binaria + alinhar tipos permitidos com a especificacao.

4. Media
Conclusao de pane inconsistente
- Arquivo: app\panes\service.py
- PUT /panes/{id} pode levar a RESOLVIDA sem preencher concluido_por_id.
- Risco: perda de auditoria e rastreabilidade.
- Fix: forcar /concluir para resolver ou registrar conclusor tambem no PUT.

5. Media
Filtro por data nao exposto na API
- Arquivos: app\panes\service.py, app\panes\router.py
- Service suporta data_inicio/data_fim, router nao.
- Risco: RF-06 parcialmente implementado.
- Fix: adicionar parametros de data no endpoint GET /panes/.

6. Media
Montagem de uploads depende da ordem de inicializacao
- Arquivo: app\main.py
- Diretorio e criado no startup, mas o mount ocorre antes.
- Risco: ambiente limpo pode nao expor a rota esperada.
- Fix: criar diretorio antes do mount ou abandonar rota estatica.

7. Media
Codigos HTTP inconsistentes
- Arquivos: routers de aeronaves, equipamentos e panes
- 404 e 409 sao usados de forma misturada para nao encontrado e conflito.
- Risco: API menos previsivel e integracao mais fragil.
- Fix: padronizar 404 para inexistencia, 409 para conflito, 422 para entrada invalida.

8. Baixa/Media
Controle transacional escondido na dependencia
- Arquivo: app\dependencies.py
- get_db faz commit automatico no fim da requisicao.
- Risco: fronteira transacional implícita e manutencao mais dificil.
- Fix: explicitar melhor a politica transacional ou documentar formalmente.

9. Baixa/Media
Desalinhamento entre codigo e especificacao
- Exemplos:
- upload aceita PDF, SPECS fala jpg/png
- API aceita observacao_conclusao, mas nao persiste
- comentarios de regra de edicao nao estao totalmente consistentes
- Fix: alinhar docs, contratos e implementacao.

10. Baixa
Cobertura de testes insuficiente para riscos criticos
- Faltam testes para:
- protecao de POST /auth/usuarios
- acesso indevido a anexos
- garantia de concluido_por_id
- filtro por data
- validacao real de tipo de arquivo

Prioridade de correcao
1. Proteger criacao de usuarios.
2. Remover exposicao publica de uploads.
3. Endurecer validacao de upload.
4. Unificar fluxo de conclusao de panes.
5. Expor filtro por data.
6. Padronizar erros HTTP.
7. Reforcar testes.

Observacao
- Houve tentativa de execucao da suite de testes, mas o ambiente expirou por tempo. Este resumo reflete revisao tecnica direta do codigo.
